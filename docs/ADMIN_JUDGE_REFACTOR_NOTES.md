# Admin / Judge 重构说明

## Admin 布局重构
- 布局改为左侧固定导航 + 右侧主内容滚动区。
- 左侧导航宽度固定在 `280px`，满足 `250-300px` 规范。
- 右侧包含面包屑与标题栏，内容区 `overflow-y-auto`，解决长列表无法滚动问题。
- 在 `1200px+` 桌面宽度下展示完整双栏结构；小屏自动收敛为单栏。

## 角色认证分离
- 新增独立登录入口：
  - `/admin/login`
  - `/judge/login`
- 登录请求加入 `portal` 参数，后端按角色与入口匹配校验。
- 中间件新增路由守卫：
  - Admin 路由仅 ADMIN 可访问；
  - Judge 路由仅 JUDGE 可访问；
  - API `/api/admin/*`、`/api/judge/*` 也做角色拦截。

## Judge 极简界面
- 采用白底卡片布局，每页最多 9 项分配任务。
- 顶部固定进度条显示已完成/总项目数。
- 提供评分标准折叠面板，便于快速查阅。
- 支持一键进入评分页面。

## 评分交互能力
- 评分页支持：
  - 每 30 秒自动暂存（本地 + 服务端）；
  - 手动暂存；
  - 最终提交确认弹窗；
  - 批量提交暂存评分。
- 关键评分数据离线缓存：localStorage（按 assignment 粒度）。

## 安全与审计
- API 权限校验统一为 `requireApiRole`。
- 评分提交加入归属校验（judgeId 必须匹配）。
- 关键动作写审计日志（分配、晋级、暂存、提交、批量提交）。
