# 首页交互与布局测试方案（V1）

## 1. 测试目标
- 验证首页无异常滑动（横向与异常纵向）。
- 验证滚动过程中关键交互元素不被遮挡且可点击。
- 验证 fixed Footer 始终固定底部并与内容区不冲突。
- 验证多设备、多浏览器下视觉与行为一致。

## 2. 测试环境矩阵
- 设备宽度：`320`、`375`、`768`、`1024`、`1440`、`1920`。
- 浏览器：Chrome、Safari、Edge。
- 系统：macOS、iOS、Android。
- 网络条件：正常网络 + 低速网络（验证首屏稳定性与抖动）。

## 3. 功能测试用例
- 首屏展示：首次进入 `/`，主标题、副标题、CTA、时间、奖池、Footer 全部可见。
- 主链路跳转：点击“提交项目”进入 `/submit`；点击“查看详情”进入 `/docs`。
- 辅助链路跳转：点击 Footer 的隐私与条款链接可正常访问。
- 键盘可达：Tab 顺序覆盖导航、首页按钮、Footer 链接，focus 样式清晰。

## 4. 滚动与遮挡测试
- 无横向溢出：断点下 `scrollWidth - clientWidth < 2`。
- 固定 Footer：页面滚动前后 Footer 的 `getBoundingClientRect().top` 基本不变。
- 可点击验证：滚动到页面底部后，System Access 与 Footer 链接仍可点击。
- 锚点补偿验证：带锚点页面滚动定位后，标题和内容不被 Header/Footer 遮挡。

## 5. 兼容性与回归测试
- 视觉回归：执行 `npm run visual:test`，对首页、短页、长页、提交页进行快照比对。
- 交互回归：执行 `npm run test:overflow`，确认无溢出和固定 Footer 行为稳定。
- 手势滚动：移动端实机验证上下滑动、快速滑动、惯性滑动无异常回弹。
- 缩放场景：系统字体放大与浏览器缩放下，关键按钮仍在可视范围。

## 6. 用户行为路径测试
- 路径 A：首页 -> 提交页 -> 返回首页 -> 查看文档。
- 路径 B：首页下滑浏览 -> 点击 Footer 链接 -> 返回首页。
- 路径 C：键盘用户仅通过 Tab 完成“进入首页并发起一次主操作”。
- 路径 D：移动端单手操作，验证底部固定 Footer 不干扰主要按钮点击。

## 7. 验收标准
- 无横向滑动异常，纵向滚动行为稳定。
- 任意断点下所有关键元素无遮挡、可点击、可聚焦。
- Footer 始终固定底部，内容区不会被 Footer 覆盖。
- 自动化测试与视觉回归通过，无新增严重控制台错误。
