# Security & HA Blueprint

## 1. 目标与SLO

- 可用性：99.9%
- 关键接口 P95 响应时间：<200ms
- 可扩展性：水平扩展到当前 10x 流量
- 容灾目标：RPO < 15 分钟，RTO < 30 分钟
- 合规目标：满足等保三级核心控制项

## 2. 多层安全防护体系

### 2.1 网络安全

- 入口架构：CDN + WAF + L7 负载均衡 + 私有子网服务
- 防火墙策略：
  - 公网仅放行 443
  - 管理平面仅允许 VPN 网段访问
  - 数据库、Redis、MQ 仅开放内网安全组
- DDoS 防护：
  - 启用云厂商高防与速率限制
  - 针对 `/api/projects` 单独配置突发保护
- VPN 访问控制：
  - 运维与管理员统一走 MFA VPN
  - 按角色分配最小权限网络段

### 2.2 应用安全

- 已落地：
  - 统一输入校验与 sanitization（Zod + 服务器端清洗）
  - Origin 校验防护跨站请求伪造
  - 全局限流（含公开提交接口独立阈值）
  - 安全响应头与 CSP
  - 文档 iframe 域名白名单校验
- 待补强：
  - 管理接口引入双提交 Cookie CSRF Token
  - 登录失败锁定与验证码策略
  - 上传类接口增加 MIME 与文件内容扫描

### 2.3 数据安全

- 存储：
  - 敏感字段分级：公开、内部、敏感、严格敏感
  - 严格敏感数据使用列级加密（AES-256-GCM）
- 密钥管理：
  - 统一接入 KMS，按环境分层密钥
  - 每 90 天轮换，支持版本化与回滚
- 数据脱敏：
  - 日志只保留必要字段，邮箱手机号掩码
  - 分析报表使用脱敏视图
- 传输加密：
  - 全链路 TLS 1.3
  - 服务间通信启用 mTLS

### 2.4 身份认证

- 已落地：
  - JWT 会话与 RBAC 鉴权
  - 密码哈希校验（scrypt）并保留旧数据兼容
  - 会话有效期收敛（8小时）
- 待补强：
  - 管理员与评委强制 MFA
  - refresh token 轮换与吊销列表
  - 权限最小化审查与季度复核

## 3. 高可用与高性能架构

### 3.1 负载均衡与弹性

- 使用云 L7 负载均衡，健康检查 `/api/health`
- 应用容器化部署，最少 3 副本，跨可用区
- 自动扩缩容依据 CPU、RT、QPS
- 压测目标：支持 100k 并发连接，按连接池与文件描述符配额调优

### 3.2 数据层

- 数据库：
  - 切换 PostgreSQL 主从架构
  - 读写分离：写入主库、查询走只读副本
  - 分库分表：按赛事/时间维度分片
- Redis：
  - 部署 Redis Cluster + 哨兵
  - 缓存热点：比赛配置、榜单、统计聚合
  - 统一 TTL 与失效回源机制

### 3.3 异步解耦

- MQ 削峰填谷：
  - 邮件通知、导出任务、审计归档改为异步消费
  - 重试、退避、死信队列、幂等键
- 服务保护：
  - 熔断、隔离、超时、退避重试
  - 服务降级：高峰期关闭非核心查询与重计算接口

### 3.4 静态资源与边缘加速

- CDN 缓存静态资源与国际化文本
- 对公共页面启用边缘缓存与压缩
- 使用版本化资源和缓存失效策略

## 4. 监控与告警体系

### 4.1 实时威胁检测

- SIEM 聚合：
  - WAF 日志、应用审计日志、系统日志、数据库审计日志
- 异常行为分析：
  - 爆破登录、异常地域访问、批量评分异常、突发提交峰值

### 4.2 性能监控

- 指标：
  - QPS、RT(P50/P95/P99)、错误率、超时率
  - CPU、内存、连接池、队列积压、缓存命中率
- 已落地基础接口：
  - `/api/health`
  - `/api/metrics`
  - 关键 API 的结构化性能日志

### 4.3 自动化告警

- 多渠道：飞书/Slack + 邮件 + 值班电话
- 告警分级：
  - P1：全站不可用或核心交易失败
  - P2：性能严重退化、错误率异常
  - P3：容量预警与安全可疑行为

## 5. 应急响应与备份恢复

- 事件处置流程：
  - 发现 → 分级 → 隔离 → 取证 → 修复 → 复盘
- 备份策略：
  - 数据库全量每日、增量每 5 分钟
  - 关键对象存储跨区域复制
- 恢复演练：
  - 月度恢复演练验证 RPO/RTO
  - 双活切换与回切脚本定期演练
- 审计留存：
  - 安全与操作审计日志留存 6 个月以上

## 6. 等保三级对齐

- 身份鉴别：MFA、账号分级、定期口令轮换
- 访问控制：最小权限、网络分区、白名单
- 安全审计：统一日志中心、关键操作可追溯
- 入侵防范：WAF、主机加固、漏洞扫描、补丁管理
- 数据完整性与保密性：TLS1.3、加密存储、密钥托管
- 灾难恢复：跨区容灾、备份恢复演练、应急预案

## 7. 分阶段落地顺序

- Phase A：应用层安全与观测增强
- Phase B：数据库迁移与读写分离、Redis 集群、MQ 异步化
- Phase C：多可用区部署、自动扩缩容、全链路压测
- Phase D：等保三级测评、渗透测试、整改闭环
